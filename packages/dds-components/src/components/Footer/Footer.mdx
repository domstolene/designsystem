`<Footer>` er en fleksibel komponent som tillatter å bygge en footer fra bunn av.
For å bygge layout brukes komponentene `<Grid>` og `<GridChild>` sammen med Footer-subkomponenter, som sørger for fritt struktur og gjennomgående design på tvers av implementasjoner.
Via bruk av `<Grid>` kan du enkelt sette brekkpunkter for innholdet.

Følgende subkomponenter er tilgjengelige:

- `<Footer>` - selve `<footer>`. Bruker mørk bagrunn og sørger for at alle tekster og ikoner brukt via Elsa-komponenter har kontrasterende farge.
- `<FooterLogo>` - logo til domstolene. plasseres til venstre.
- `<FooterList>` - liste med elementer, sørger for riktig spacing og layout.
- `<FooterSocialsList>` - egen komponent for liste med lenker til sosiale medier; til bruk under Logo. Sørger for riktig spacing og layout.
- `<FooterListHeader>` - overskrift for en liste; kan brukes for alle lister.
- `<FooterLeft>` - venstre seksjonen i footer; Kan brukes som container for logo og liste med sosiale medier. Sørger for riktig layout.
- `<FooterListGroup>` - layout-komponent for `<FooterListHeader>` og `<FooterList>`.
- `<FooterSocialsListGroup>` - layout-komponent for `<FooterListHeader>` og `<FooterSocialsList>`.

## Retningslinjer

### `row-gap` i `<Grid>` for liten skjerm

Default `row-gap` i `<Grid>` vil være for liten når `<GridChild>` tar hele bredden, aka versjon for liten skjerm.
For disse brekkpunktene bør du ha `row-gap` satt til `dds-spacing-x2-5`.

<Source
  code={`
<Grid
  rowGap={{
      xs: 'var(--dds-spacing-x2-5)', // liten skjerm
      sm: 'var(--dds-spacing-x2-5)' // liten skjerm
    }}
 >
  <GridChild
  columnsOccupied={{
        xs: '1/-1' // liten skjerm
        sm: '1/-1', // liten skjerm
        md: '1/4',
        lg: '1/4',
        xl: '1/4',
      }}
  >
   {...innhold}
  </GridChild>
   {...mer innhold}
</Grid>
`}
/>
